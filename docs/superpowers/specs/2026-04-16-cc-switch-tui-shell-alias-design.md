# cc-switch-tui Shell 别名集成设计文档

## 1. 目标

让 cc-switch-tui 不仅为每个 provider 实例保存配置，还能为其生成带 `cl-` 前缀的 shell 别名。同时支持将一个实例设为"默认激活"，此时普通 `claude` 命令指向该实例。所有别名写入 `~/.cc-switch-tui/aliases.zsh`，TUI 自动确保 `~/.zshrc` 已加载该脚本。

## 2. 核心设计原则

- **别名必填**：每个实例在创建时必须输入别名（强制以 `cl-` 为前缀）
- **eval-safe**：生成的 `aliases.zsh` 只包含 `export` 和 `alias` 命令，可被 `source`
- **默认激活**：列表页按 `Enter` 激活当前实例，激活后 `claude` 命令指向该实例
- **自动注入**：TUI 启动时检查 `~/.zshrc`，缺少 `source` 行则自动追加，并底部提示用户

## 4. 数据模型变更

### 4.1 ProviderInstance 新增 alias

```rust
pub struct ProviderInstance {
    pub id: String,
    pub template_id: String,
    pub model_id: String,
    pub api_key: String,
    pub created_at: DateTime<Utc>,
    pub alias: String,        // 新增：用户输入的别名，如 "cl-mini"
}
```

### 4.2 SQLite 表 Migration

`instances` 表增加 `alias TEXT NOT NULL DEFAULT ''` 列。

现有数据（`alias = ''`）在 TUI 中显示为"无别名"，用户必须补填后才能使用激活/别名功能。

## 5. 别名生成机制（aliases.zsh）

`shell.rs` 中的核心函数：

```rust
/// 根据所有实例和当前激活实例，重写 aliases.zsh
pub fn generate_aliases(
    instances: &[ProviderInstance],
    templates: &[ProviderTemplate],
    current_instance_id: Option<&str>,
) -> std::io::Result<()>
```

生成的 `~/.cc-switch-tui/aliases.zsh` 示例：

```zsh
# Auto-generated by cc-switch-tui. Do not edit manually.
alias cl-mini='ANTHROPIC_BASE_URL=https://api.minimaxi.com/anthropic ANTHROPIC_AUTH_TOKEN=sk-xxx ANTHROPIC_MODEL=MiniMax-M2.7-highspeed API_TIMEOUT_MS=3000000 CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1 claude'
alias cl-opus='ANTHROPIC_BASE_URL=... ... claude'
alias claude='ANTHROPIC_BASE_URL=... ... claude'
```

规则：
1. 每个有 `alias` 的实例生成一条 `alias <alias>='... claude'`
2. 如果存在 `current_instance_id`，额外追加 `alias claude='... claude'`
3. 环境变量合并顺序：`template.default_env` → `model.env_overrides` → `ANTHROPIC_AUTH_TOKEN`

## 6. zshrc 自动注入

```rust
/// 检查 ~/.zshrc 是否包含 source ~/.cc-switch-tui/aliases.zsh，没有则追加
pub fn ensure_zshrc_source() -> std::io::Result<bool>
```

行为：
- 如果 `~/.zshrc` 不存在，创建它
- 如果已包含 `source ~/.cc-switch-tui/aliases.zsh`（忽略前导空白），跳过
- 否则在文件末尾追加一行：
  ```zsh
  source ~/.cc-switch-tui/aliases.zsh
  ```
- 返回 `true` 表示本次进行了修改，`false` 表示无需修改

TUI 启动时（`main.rs`）先调用 `ensure_zshrc_source()`，如果返回 `true`，在底部帮助栏上方显示一行临时提示：
`已自动配置 ~/.zshrc，请执行 source ~/.zshrc 生效`

## 7. TUI 流程变更

### 7.1 创建向导增加 Alias 输入页

当前状态机增加：

```rust
pub enum AppState {
    // ... existing states ...
    CreateAlias { template_id: String, model_id: String, api_key: String },
}
```

流程变为：
1. `CreateProvider` → `CreateModel` → `CreateApiKey` → `CreateAlias` → 提交创建

`CreateAlias` 页面与 `CreateApiKey` 类似，输入框显示 `cl-` 前缀作为占位符或自动补全。用户输入 `mini` 后，系统自动补全为 `cl-mini`。

**别名校验规则：**
- 必须以 `cl-` 开头
- 只能包含 `a-z`, `A-Z`, `0-9`, `-`, `_`
- 不能为空
- 不能与已有实例的 alias 重复

校验失败时显示红色错误弹窗，不退出输入页。

### 7.2 列表页新增操作

帮助栏更新为：
```
↑↓:移动  Enter:激活  n:新建  a:编辑别名  e:编辑Key  d:删除  q:退出
```

- **`Enter`**：激活当前实例
  - 设置 `current_instance_id`
  - 调用 `shell::generate_aliases()`
  - 底部显示临时提示：`已激活 <alias>，新终端中 claude 命令将使用该配置`

- **`a`**：编辑别名
  - 弹出单行输入框（类似编辑 API Key）
  - 修改后更新数据库并重新生成 `aliases.zsh`

### 7.3 无别名实例的处理

如果当前高亮实例的 `alias` 为空字符串（旧数据）：
- 按 `Enter` 尝试激活时，弹出错误提示：`请先按 a 设置别名`
- 按 `a` 正常弹出别名编辑框
